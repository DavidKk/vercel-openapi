# Context

Aggregated context for AI: minimal load to understand "what is this project" and "what am I doing now". Reduces token use; drill into specs/rules/tasks as needed.

---

| File           | Purpose                                                                              |
| -------------- | ------------------------------------------------------------------------------------ |
| **current.md** | Current focus: link to active task(s) and key spec/rule. Update when switching work. |
| **summary.md** | One-page project and architecture summary.                                           |

Load order for AI: `context/current.md` → then spec / rules / task per "Do X → read Y".
