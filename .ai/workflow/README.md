# Workflow

Process definitions for AI and humans. **Read the relevant workflow before starting the task.** See **`.ai/INDEX.md`** for which workflow file applies to your task.

| Workflow               | When to use                                                                                                                                                       |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Requirements audit** | **Mandatory** before new modules, new public `/api` behavior, or material features. Read `workflow/requirements-audit.md` first; get explicit developer approval. |
| **Module development** | After audit passes (or spec-first docs are agreed): read `workflow/module-development.md` and follow the five phases in order.                                    |

New workflows (e.g. API change, release) can be added here and under `workflow/<name>.md`.
